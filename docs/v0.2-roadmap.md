# Roadmap — v0.2 and beyond

v0.2 is the structured planning and contract layer, built *before* any runtime so the future harness enforces rules that were proven by hand first.

| Version | Theme | What it adds |
|---------|-------|--------------|
| **v0.2** | Structured orchestration docs and templates | Task classes, capability records, frontmatter brief + run report contract, issue templates. No runtime code. |
| **v0.3** | Local dispatch harness | A thin runner that parses brief frontmatter, creates the worktree, dispatches, collects the run report. |
| **v0.4** | Executor management | Warm OpenCode server adapter, executor health/status tracking, smarter availability fallback. |
| **v0.5** | Parallel attempts | Multiple executors on the same brief, judge comparison, best-of-N integration. |

## Deferred to v0.3+

- **Runtime/harness code** of any kind (`orchestrate run`, automatic worktree creation, automatic dispatch, automatic patch application, GitHub Actions, parallel runtime, model API calls, npm packaging).
- **Run history persistence.** v0.2 standardizes only the on-disk layout (`.orchestrate/runs/<runId>/` with report, diff, logs). v0.3 should evaluate OpenCode's local storage approach (https://github.com/anomalyco/opencode) as the persistence layer for run history before designing one from scratch.
- **YAML issue forms** (`.github/ISSUE_TEMPLATE/*.yml`). Markdown templates ship in v0.2; forms would enforce required fields and are worth the migration once the field set stabilizes.
- **OpenCode paid models** (`gpt-5.5` via OpenCode). Deferred until the Zen workspace is funded and cold-start cost is addressed; until then GPT-5.5-class advisory work routes through codex read-only.
